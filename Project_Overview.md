# Архитектура и логика работы проекта automatizationCalc (RocketApp)

## 1. Общее описание проекта
**automatizationCalc** — это десктопное инженерное приложение для параметрического проектирования, прочностного анализа и весовой оптимизации подкрепленных конических и цилиндрических оболочек (например, отсеков ракет или фюзеляжей). 

Программа автоматизирует полный цикл расчетов: генерацию геометрии, построение конечно-элементной сетки (КЭ-сетки) в **Siemens Femap**, настройку и запуск оптимизатора **MSC Nastran (SOL 200)**, обновление модели по результатам оптимизации и последующий поверочный расчет на устойчивость **(SOL 105)**.

## 2. Технологический стек
*   **Язык программирования:** Python 3.
*   **GUI Фреймворк:** PySide6 (Qt for Python).
*   **CAE/FEA Интеграция:** Siemens Femap (через Windows COM API), MSC Nastran.
*   **База данных:** PostgreSQL (для сохранения результатов итераций).
*   **Хранение конфигураций:** JSON (`config.json`, `profiles.json`, `checkpoint.json`).

## 3. Программная архитектура (Паттерны проектирования)
Проект спроектирован с использованием четкого разделения ответственности (Separation of Concerns).

### 3.1. Паттерн MVC (Model-View-Controller)
*   **View (Представление):** Файлы `ui/main_window.py` и `ui_main_window.py`. Отвечают за отображение интерфейса, сбор геометрических параметров конуса, параметров материалов, нагрузок и диапазонов для оптимизации.
*   **Controller (Контроллер):** Класс `RocketController` (`controller.py`). Выступает связующим звеном. Принимает сигналы от UI, формирует пакеты данных (словарь `params`) и управляет запуском рабочего потока. Отвечает за логику создания точек сохранения (checkpoints).
*   **Model (Модель/Логика):** Фоновый поток `OptimizationWorker` (`worker.py`), который управляет инженерной логикой и расчетами, не блокируя пользовательский интерфейс.

### 3.2. Паттерн Director-Builder (Директор - Строитель)
Управление процессом построения модели в Femap вынесено в классический паттерн Builder:
*   **Director (`director.py`):** Класс `OptimizationDirector` содержит жестко заданный алгоритм (пайплайн) расчета `construct_and_solve()`. Он знает *в каком порядке* нужно вызывать методы, но не знает *как* они работают.
*   **Builder (`builders.py`):** Класс `ConeModelBuilder` содержит реализацию каждого шага для Femap. Он знает *как* создать материал, *как* построить геометрию `feSolidPrimitive`, *как* создать сетку `feMeshSurface2` и приложить нагрузки через жесткие элементы RBE2.

## 4. Инженерная логика и рабочий процесс (Pipeline)
Рабочий процесс запускается в фоновом потоке и итерируется по различным конфигурациям (разное количество стрингеров, разные типы профилей из `profiles.json`).

Один цикл вычисления (внутри `director.py`) состоит из следующих этапов:

1.  **Инициализация модели:** Подключение к сессии Femap, очистка рабочей области.
2.  **Построение КЭ-модели:**
    *   Создание материалов (Material) и свойств (Property) для обшивки (Plate) и стрингеров (Beam).
    *   Генерация геометрии конуса/цилиндра.
    *   Генерация КЭ-сетки обшивки.
    *   Размещение балочных элементов (стрингеров Z-образного сечения) вдоль образующих конуса.
    *   Склейка узлов (`merge_nodes`) и создание жестких связей на торцах для корректной передачи нагрузки.
    *   Приложение граничных условий (нагрузки и закрепления).
3.  **Оптимизация массы (Nastran SOL 200):**
    *   Формирование входного `.dat` файла.
    *   Внедрение карт оптимизации с помощью шаблона `nastran_opt_template.txt`. Целевая функция (Objective) — минимизация массы (Weight). Варьируемый параметр (Design Variable) — толщина обшивки. Ограничения (Constraints) — допускаемые напряжения по Мизесу и минимально допустимый коэффициент потери устойчивости (Eigenvalue > 1.3).
    *   Запуск внешнего процесса Nastran.
4.  **Обновление модели:**
    *   Парсинг файлов результатов (`.pch`, `.f06`) для получения найденной оптимальной толщины обшивки.
    *   Обновление свойств (`Property`) в текущей модели Femap с новой толщиной.
5.  **Поверочный расчет (Nastran SOL 105):**
    *   Запуск линейного расчета на устойчивость обновленной конструкции.
    *   Чтение финальных результатов (масса, коэффициенты запаса).
6.  **Аналитика и сохранение:**
    *   Расчет локальных коэффициентов запаса (Margins of Safety) по прочности обшивки, стрингеров и местной устойчивости полок стрингеров внутри `worker.py`.
    *   Запись результатов итерации в базу данных PostgreSQL (`db_manager.py`).

## 5. Интеграция с подсистемами

### 5.1. Взаимодействие с Femap
Осуществляется через Windows COM API. 
*   `session.py` содержит Singleton для поддержания единственного подключения к Femap (`win32com.client.Dispatch("femap.model")`).
*   `Pyfemap.py` предоставляет Python-обертки для констант и типов данных Femap.

### 5.2. Управление состоянием (Checkpoints)
Из-за длительности оптимизационных расчетов внедрена система восстановления. Состояние перебора параметров постоянно сохраняется в `checkpoint.json`. В случае падения программы или экстренной остановки, при следующем запуске Контроллер предложит продолжить расчет с прерванной итерации.
